# GoBuyMe — Master Build Prompt
> Use this prompt as the system context for any AI coding session (Claude, Cursor, Copilot, etc.).
> It contains the full product specification, technical architecture, design system, and coding standards
> required to build GoBuyMe to enterprise grade.

---

## 1. Project Identity

**App name:** GoBuyMe
**Tagline:** Hungry? GoBuyMe. Anything.
**Company:** Bubble Barrel Commerce Limited
**Headquarters:** Owerri, Nigeria
**Website:** https://gobuyme.shop
**Platform:** iOS + Android (React Native / Expo)
**Type:** On-demand delivery super-app for the Nigerian market

GoBuyMe is a single mobile app that serves three distinct user types — Customers, Vendors, and Riders —
selected at onboarding. It delivers food, groceries, pharmacy items, and errands across Nigerian cities
in 25 minutes or less. It is not three separate apps; it is one unified app with role-based navigation.

### Typography
**Font:** Plus Jakarta Sans (Google Font)
- Import weights: 400 (Regular), 500 (Medium), 600 (SemiBold), 700 (Bold), 800 (ExtraBold)
- Heading 1: 32px / ExtraBold / -0.5 letter-spacing
- Heading 2: 26px / Bold
- Heading 3: 20px / Bold
- Heading 4: 17px / SemiBold
- Body Large: 16px / Regular / 1.6 line-height
- Body: 14px / Regular / 1.5 line-height
- Caption: 12px / Medium
- Label: 11px / SemiBold / 0.5 letter-spacing / UPPERCASE

### Design Language
Bold and expressive — large typography, vibrant colors, high contrast.
Reference aesthetic: Glovo. Every screen should feel energetic and confident.

### Component Principles
- Border radius: 4px (inputs), 8px (cards, buttons), 12px (bottom sheets), 16px (modals), 999px (pills/chips)
- Elevation: Use shadow sparingly — one level per surface layer
- Spacing scale: 4, 8, 12, 16, 20, 24, 32, 40, 48, 64 (multiples of 4)
- Icons: react-native-vector-icons (Feather set as primary, Ionicons as secondary)
- Haptic feedback on all primary CTAs
- Skeleton loaders — never blank states during loading
- Empty states — always illustrated, never just text

## 3. Tech Stack

### Mobile App
```
Framework:       React Native 0.74+ (Expo SDK 51+)
Language:        JavaScript (JSX) — no TypeScript for now, but structure for easy migration
Navigation:      React Navigation v6 (Stack + Bottom Tab + Drawer)
State:           Redux Toolkit + RTK Query (for server state)
Styling:         StyleSheet API — no styled-components; use a central theme object
Font loading:    expo-google-fonts/plus-jakarta-sans
Maps:            react-native-maps + Google Maps API
Location:        expo-location
Notifications:   expo-notifications
Camera/Image:    expo-image-picker
Auth (Google):   expo-auth-session + Google OAuth
Storage:         expo-secure-store (tokens), @react-native-async-storage (non-sensitive)
Real-time:       socket.io-client
Payments:        Paystack React Native SDK (@paystack/react-native-sdk)
Error tracking:  @sentry/react-native
i18n:            i18next + react-i18next (English first)
HTTP:            Axios (with interceptors for JWT refresh)
```

### Backend
```
Runtime:         Node.js 20+
Framework:       Express.js
Language:        JavaScript (CommonJS)
ORM:             Prisma 5+
Database:        PostgreSQL 15+ (local via pgAdmin; cloud on Railway or Supabase later)
Auth:            JWT (access token: 15min, refresh token: 30 days) + Google OAuth (passport-google-oauth20)
Real-time:       Socket.io 4+
Payments:        Paystack Node.js SDK
Email:           Nodemailer (SMTP — Brevo/Resend for prod)
Push:            Expo Server SDK (@expo/server-sdk)
Validation:      Joi (route-level schema validation)
Security:        helmet, cors, express-rate-limit, bcryptjs (salt rounds: 12)
Logging:         winston (file + console), morgan (HTTP)
Error tracking:  @sentry/node
Env:             dotenv
File uploads:    multer + Cloudinary
```

### Admin Dashboard (Web — separate project)
```
Framework:       React (Vite)
UI Library:      Shadcn/ui + Tailwind CSS
Charts:          Recharts
Auth:            Same JWT system — SUPER_ADMIN role guard
State:           Zustand
```

## 5. Database Schema (Prisma)

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ─── Enums ───────────────────────────────────────────────────────────────────

enum Role {
  CUSTOMER
  VENDOR
  RIDER
  SUPER_ADMIN
}

enum VendorCategory {
  RESTAURANT
  GROCERY
  PHARMACY
  ERRAND
}

enum ApprovalStatus {
  PENDING
  APPROVED
  REJECTED
  SUSPENDED
}

enum OrderStatus {
  PENDING        // order placed, awaiting vendor confirmation
  CONFIRMED      // vendor accepted
  PREPARING      // vendor is preparing
  READY          // ready for pickup by rider
  PICKED_UP      // rider collected
  IN_TRANSIT     // rider on the way
  DELIVERED      // completed
  CANCELLED      // cancelled by any party
}

enum PaymentStatus {
  PENDING
  PAID
  FAILED
  REFUNDED
}

enum PaymentMethod {
  CARD
  BANK_TRANSFER
  PAYSTACK_USSD
  CASH_ON_DELIVERY
}

// ─── Models ──────────────────────────────────────────────────────────────────

model User {
  id                String       @id @default(uuid())
  name              String
  email             String       @unique
  phone             String?      @unique
  password          String?      // null for OAuth users
  googleId          String?      @unique
  role              Role         @default(CUSTOMER)
  avatar            String?
  isEmailVerified   Boolean      @default(false)
  isPhoneVerified   Boolean      @default(false)
  isActive          Boolean      @default(true)
  refreshToken      String?
  pushToken         String?      // Expo push token
  preferredLocale   String       @default("en")
  createdAt         DateTime     @default(now())
  updatedAt         DateTime     @updatedAt

  customer          Customer?
  vendor            Vendor?
  rider             Rider?
  auditLogs         AuditLog[]
}

model Customer {
  id        String     @id @default(uuid())
  userId    String     @unique
  user      User       @relation(fields: [userId], references: [id], onDelete: Cascade)
  addresses Address[]
  orders    Order[]
  favorites Favorite[]
  cart      Cart?
  createdAt DateTime   @default(now())
}

model Vendor {
  id              String         @id @default(uuid())
  userId          String         @unique
  user            User           @relation(fields: [userId], references: [id], onDelete: Cascade)
  businessName    String
  slug            String         @unique  // URL-friendly name
  description     String?
  logo            String?
  coverImage      String?
  category        VendorCategory
  address         String
  city            String
  state           String         @default("Rivers")
  latitude        Float?
  longitude       Float?
  isOpen          Boolean        @default(false)
  approvalStatus  ApprovalStatus @default(PENDING)
  autoApprove     Boolean        @default(false) // flag for future auto-approval
  commissionRate  Float          @default(10.0)  // % platform takes
  rating          Float          @default(0)
  totalRatings    Int            @default(0)
  openingTime     String?        // e.g. "08:00"
  closingTime     String?        // e.g. "22:00"
  menuItems       MenuItem[]
  orders          Order[]
  payoutAccount   PayoutAccount?
  createdAt       DateTime       @default(now())
  updatedAt       DateTime       @updatedAt
}

model Rider {
  id              String    @id @default(uuid())
  userId          String    @unique
  user            User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  vehicleType     String    // bike, bicycle, car
  plateNumber     String?
  isAvailable     Boolean   @default(false)
  isOnline        Boolean   @default(false)
  latitude        Float?
  longitude       Float?
  approvalStatus  ApprovalStatus @default(PENDING)
  rating          Float     @default(0)
  totalRatings    Int       @default(0)
  deliveries      Order[]
  earnings        Earning[]
  payoutAccount   PayoutAccount?
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
}

model MenuItem {
  id          String      @id @default(uuid())
  vendorId    String
  vendor      Vendor      @relation(fields: [vendorId], references: [id], onDelete: Cascade)
  name        String
  description String?
  price       Float
  image       String?
  category    String?
  isAvailable Boolean     @default(true)
  isFeatured  Boolean     @default(false)
  cartItems   CartItem[]
  orderItems  OrderItem[]
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt
}

model Cart {
  id         String     @id @default(uuid())
  customerId String     @unique
  customer   Customer   @relation(fields: [customerId], references: [id], onDelete: Cascade)
  vendorId   String?    // cart locked to one vendor at a time
  items      CartItem[]
  updatedAt  DateTime   @updatedAt
}

model CartItem {
  id         String   @id @default(uuid())
  cartId     String
  cart       Cart     @relation(fields: [cartId], references: [id], onDelete: Cascade)
  menuItemId String
  menuItem   MenuItem @relation(fields: [menuItemId], references: [id])
  quantity   Int      @default(1)
  note       String?
}

model Address {
  id         String   @id @default(uuid())
  customerId String
  customer   Customer @relation(fields: [customerId], references: [id], onDelete: Cascade)
  label      String   // Home, Work, Other
  address    String
  city       String
  state      String   @default("Rivers")
  latitude   Float?
  longitude  Float?
  isDefault  Boolean  @default(false)
  createdAt  DateTime @default(now())
}

model Order {
  id                String        @id @default(uuid())
  orderNumber       String        @unique  // e.g. GBM-20260419-0001
  customerId        String
  customer          Customer      @relation(fields: [customerId], references: [id])
  vendorId          String
  vendor            Vendor        @relation(fields: [vendorId], references: [id])
  riderId           String?
  rider             Rider?        @relation(fields: [riderId], references: [id])
  items             OrderItem[]
  status            OrderStatus   @default(PENDING)
  subtotal          Float
  deliveryFee       Float
  platformFee       Float         // commission deducted
  totalAmount       Float
  deliveryAddress   String
  deliveryLatitude  Float?
  deliveryLongitude Float?
  distanceKm        Float?
  paymentMethod     PaymentMethod
  paymentStatus     PaymentStatus @default(PENDING)
  paystackRef       String?
  paystackVerified  Boolean       @default(false)
  note              String?
  estimatedTime     Int?          // minutes
  cancelReason      String?
  rating            Int?          // 1-5, set after delivery
  review            String?
  createdAt         DateTime      @default(now())
  updatedAt         DateTime      @updatedAt
}

model OrderItem {
  id         String   @id @default(uuid())
  orderId    String
  order      Order    @relation(fields: [orderId], references: [id], onDelete: Cascade)
  menuItemId String
  menuItem   MenuItem @relation(fields: [menuItemId], references: [id])
  name       String   // snapshot at time of order
  price      Float    // snapshot at time of order
  quantity   Int
}

model Favorite {
  id         String   @id @default(uuid())
  customerId String
  customer   Customer @relation(fields: [customerId], references: [id], onDelete: Cascade)
  vendorId   String
  createdAt  DateTime @default(now())

  @@unique([customerId, vendorId])
}

model Earning {
  id        String   @id @default(uuid())
  riderId   String
  rider     Rider    @relation(fields: [riderId], references: [id])
  orderId   String   @unique
  amount    Float
  isPaid    Boolean  @default(false)

}

model PayoutAccount {
  id            String  @id @default(uuid())
  vendorId      String? @unique
  vendor        Vendor? @relation(fields: [vendorId], references: [id])
  riderId       String? @unique
  rider         Rider?  @relation(fields: [riderId], references: [id])
  bankName      String
  accountNumber String
  accountName   String
  paystackRecipientCode String?
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}

model Offer {
  id          String   @id @default(uuid())
  title       String
  description String?
  imageUrl    String?
  code        String?  @unique
  discount    Float    // % or flat
  isPercent   Boolean  @default(true)
  minOrder    Float    @default(0)
  vendorId    String?  // null = platform-wide offer
  expiresAt   DateTime?
  isActive    Boolean  @default(true)
  createdAt   DateTime @default(now())
}

model AuditLog {
  id        String   @id @default(uuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  action    String   // e.g. "ORDER_CANCELLED", "VENDOR_APPROVED"
  entity    String   // e.g. "Order", "Vendor"
  entityId  String
  meta      Json?    // any extra context
  ip        String?
  createdAt DateTime @default(now())
}

model Notification {
  id        String   @id @default(uuid())
  userId    String
  title     String
  body      String
  type      String   // order_update, promo, system
  isRead    Boolean  @default(false)
  meta      Json?
  createdAt DateTime @default(now())
}
```

## 7. Socket.io Architecture

### Namespaces & Rooms
```
/orders   ← order status updates
/riders   ← rider location + job dispatch

Rooms:
  order:{orderId}    ← customer, vendor, rider all join when order is active
  vendor:{vendorId}  ← vendor's private room for incoming orders
  rider:{riderId}    ← rider's private room for job broadcasts
```

### Events
```
// Server → Client
order:new           ← vendor receives new order  { order }
order:status        ← all parties receive status change  { orderId, status }
rider:location      ← customer receives rider lat/lng  { riderId, lat, lng }
order:assigned      ← rider receives job offer  { order, vendor, customer }

// Client → Server
rider:updateLocation  { latitude, longitude }
order:accept          { orderId }   (rider)
order:updateStatus    { orderId, status }
```

## 8. Delivery Fee & Commission Logic

### Delivery Fee
```
Base fee:     ₦500
Per km rate:  ₦100/km
Formula:      deliveryFee = 500 + (distanceKm * 100)
Max cap:      ₦3,000
Min fee:      ₦500
```

### Platform Commission
```
Default rate:  10% of subtotal per order
Formula:       platformFee = subtotal * (vendor.commissionRate / 100)
Vendor payout: subtotal - platformFee
Rider payout:  deliveryFee * 0.85  (platform keeps 15% of delivery fee)
```

These rates are stored per-vendor on the Vendor model so they can be negotiated individually.

## 10. Auth Flow

```
Register:
1. User fills form (name, email, phone, password) + role is pre-set from onboarding
2. POST /auth/register
3. Backend creates User + role profile (Customer/Vendor/Rider)
4. Returns { accessToken, refreshToken, user }
5. Tokens stored in expo-secure-store
6. Navigate to role-specific home

Login:
1. POST /auth/login
2. Returns tokens + user + role
3. AppNavigator reads role → routes to correct navigator

Google OAuth:
1. expo-auth-session triggers Google consent
2. idToken sent to POST /auth/google
3. Backend verifies with Google, creates or finds user
4. Returns tokens (role must be confirmed if new user)

Token Refresh:
- Axios interceptor catches 401
- Calls POST /auth/refresh with stored refreshToken
- On success: retries original request with new accessToken
- On failure: clears tokens, redirects to Login
```

## 11. Enterprise-Grade Requirements

### Security
- All routes behind `verifyToken` middleware unless explicitly public
- Role guard via `requireRole(Role.VENDOR)` on all vendor routes
- Helmet.js for security headers on every response
- CORS restricted to app origin + admin domain
- Rate limiting: 100 req/15min per IP on all routes; 10 req/15min on auth routes
- Paystack webhook signature verification (HMAC-SHA512)
- Passwords hashed with bcryptjs, salt rounds: 12
- No sensitive data in JWT payload — only `{ userId, role }`
- All inputs validated with Joi before reaching controllers
- SQL injection protected by Prisma parameterised queries

### Error Handling
- Global error handler middleware catches all unhandled errors
- `catchAsync` wrapper eliminates try/catch boilerplate in controllers
- All errors logged to Sentry (`@sentry/node` on backend, `@sentry/react-native` on app)
- Client never sees raw stack traces — only clean error messages
- 4xx errors: user-facing messages
- 5xx errors: generic message + Sentry alert

### Logging
- Winston logger writes to `logs/error.log` and `logs/combined.log`
- Morgan logs all HTTP requests in development
- AuditLog model records all state-changing actions with userId, action, entity, entityId, ip

### Validation
- Every POST/PUT route has a corresponding Joi schema in `/validators`
- Validation middleware runs before the controller
- Errors returned as array: `[{ field, message }]`

### i18n
- All user-facing strings in the app go through `i18next` (`t('key')`)
- Translation files in `/i18n/locales/en.json`
- Never hardcode UI strings in JSX — always use translation keys
- Backend error messages are translation-key strings the app resolves locally

## 12. Coding Standards & Conventions

### General
- Files: PascalCase for components (`VendorCard.jsx`), camelCase for everything else
- One component per file
- No `console.log` in production — use the logger utility
- No hardcoded URLs, secrets, or magic numbers — use constants or `.env`

### React Native
- Always use `StyleSheet.create({})` — never inline style objects in JSX
- All colours from `theme.js` — never hardcode hex values in components
- All spacing from theme spacing scale
- Components receive only the data they need (no over-fetching in props)
- Use `useCallback` and `useMemo` for expensive operations and handlers in lists
- FlatList for all scrollable lists — never ScrollView with map()
- Always handle loading, error, and empty states in every screen

### Backend
- All async route handlers wrapped in `catchAsync`
- All database calls go through Prisma — no raw SQL unless absolutely necessary
- Controllers are thin — business logic lives in services
- Return `apiResponse.success(res, data, message)` or `apiResponse.error(res, message, statusCode)`
- Never return the full Prisma object — always select only required fields

### Git Conventions
```
feat:     new feature
fix:      bug fix
refactor: code restructure
style:    formatting only
chore:    config, deps, tooling
docs:     documentation only

Branch naming:
  feature/onboarding-screens
  fix/cart-quantity-bug
  refactor/auth-middleware
```
